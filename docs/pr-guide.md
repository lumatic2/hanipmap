# PR 올리는 법 (1분)

작업을 서비스에 반영해 달라고 요청하는 절차다. PR(Pull Request)은 "제 작업 확인하고 합쳐주세요"라는 뜻이다.

## 순서

1. AI 코딩 도구에 말한다: **"지금까지 작업 커밋하고 푸시해줘"**
2. 브라우저에서 github.com/lumatic2/hanipmap 접속
3. 화면 상단에 노란 배너가 떠 있으면 **"Compare & pull request"** 초록 버튼 클릭
   - 배너가 없으면: 왼쪽 위 브랜치 선택에서 본인 브랜치 선택 → "Contribute" → "Open pull request"
4. 제목에 무엇을 했는지 한 줄 적는다 (예: "사진 업로드 폼 추가")
5. **"Create pull request"** 초록 버튼 클릭. 끝.

이후는 통합 담당이 확인하고 반영한다. 수정 요청 코멘트가 달리면 같은 브랜치에서 고치고 다시 푸시만 하면 된다(PR을 새로 만들 필요 없음).

## 자주 묻는 것

- **푸시가 안 된대요**: 오류 메시지를 그대로 팀 채널에 붙여넣으면 된다. 본인이 해결하려고 붙잡지 않는다.
- **남의 파일을 실수로 고쳤어요**: 커밋 전이면 AI 도구에 "그 파일 변경 취소해줘". 이미 푸시했으면 팀 채널에 알린다. 혼나는 일 아니니 빨리 알리는 게 최선이다.
