# 한입지도

예산과 상황을 입력하면 성균관대 학식과 명륜동 주변 식당을 한 화면에 놓고 추천하는 지도 서비스.
AI Document Builders Challenge (Upstage × 대학혁신과공유센터) 출품작 · 팀 sixsense.

- 데모데이: 2026-07-25(토) · 빌드업 마감: 7/24(금)
- 팀: 김민서 · 이준범 · 정경화 · 전유성

## 진행 방식 (7/18 팀 미팅 확정)

**방향은 하나, 서비스는 각자, 데이터는 공유, 데모는 통합 담당이 보장.**

- 각자 자기 버전의 한입지도를 처음부터 끝까지 만든다. 이 레포는 **데모데이에 올라가는 대표 조립본**이다.
- 각자 버전에서 잘 나온 화면·아이디어는 PR로 제안하면 대표 버전에 편입한다.
- 공유 데이터: 구글 시트 "한입지도 식당 데이터" (팀 채널에 링크). 메뉴판 사진은 공유 드라이브 폴더에.
- 공용 API: 파싱·검색 API 주소를 제공한다(`docs/api-contract.md`). fetch 한 줄로 자기 서비스에 AI 기능을 붙일 수 있다. 자기 Upstage 키로 직접 만들어보는 것도 환영.

## 작업 흐름

1. **작업**: AI 코딩 도구(Claude Code, Codex 등)에 만들고 싶은 화면을 설명하고 자기 버전을 만든다.
2. **올리기**: 도구에 "지금까지 작업 커밋하고 푸시해줘"라고 지시한다.
3. **제안하기**: 대표 버전에 넣고 싶은 결과물이 생기면 GitHub 웹에서 PR 버튼을 누른다. 방법은 `docs/pr-guide.md` (1분 소요).

막히면 화면과 오류 메시지를 팀 채널에 올리고 다른 작업으로 전환한다. 한 문제를 혼자 3시간 이상 붙잡지 않는다.

## 하지 않는 것

- `main` 브랜치에 직접 푸시 (대표 조립본은 통합 담당이 관리)
- API 키를 코드에 붙여넣기 (키는 서버 환경변수로만)
- 다른 지도 앱 화면의 메뉴·가격 베끼기 (데이터는 실촬영 메뉴판 기준)

## 폴더 안내

```
api/       서버 함수 (Upstage 호출, 추천 계산) (대표 조립본)
public/    화면 파일
scripts/   학식 자동 수집
db/        DB 스키마
docs/      API 계약, PR 가이드
```
