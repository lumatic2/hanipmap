# 한입지도

예산과 상황을 입력하면 성균관대 학식과 명륜동 주변 식당을 한 화면에 놓고 추천하는 지도 서비스.
AI Document Builders Challenge (Upstage × 대학혁신과공유센터) 출품작 · 팀 sixsense.

- 데모데이: 2026-07-25(토) · 빌드업 마감: 7/24(금)
- 팀: 김민서 · 이준범 · 정경화 · 전유성

## 모듈, 브랜치, 작업 파일

담당은 7/18 팀 미팅에서 배정한다. 배정되면 아래 표의 본인 모듈 브랜치에서 작업한다.

| 모듈 | 브랜치 | 작업 파일 |
|---|---|---|
| A. 서버·데이터 기반 | `part-a-server` | `api/` · `db/` · `scripts/` |
| B. 메뉴 제보 | `part-b-report` | `public/report.html` |
| C. 탐색 화면 | `part-c-explore` | `public/explore.html` |
| D. 개인화 서비스 | `part-d-personal` | `public/home.html` · `api/recommend.js` · `api/events.js` |

규칙은 하나다. **본인 브랜치에서 본인 파일만 수정한다.**
파트 사이에 주고받는 데이터 형식은 `docs/api-contract.md`에 확정되어 있다. 이 규격은 개발 중에 바꾸지 않는다(변경이 필요하면 통합 담당에게 먼저).

## 작업 흐름 (3단계)

1. **작업**: AI 코딩 도구(Claude, Cursor 등)에 착수 프롬프트를 주고 본인 파일을 만든다.
2. **올리기**: 도구에 "지금까지 작업 커밋하고 푸시해줘"라고 지시한다.
3. **반영 요청**: GitHub 웹에서 PR 버튼을 누른다. 방법은 `docs/pr-guide.md` (1분 소요). 통합 담당이 확인 후 서비스에 반영한다.

막히면 화면과 오류 메시지를 팀 채널에 올리고 다른 작업으로 전환한다. 한 문제를 혼자 3시간 이상 붙잡지 않는다.

## 하지 않는 것

- 남의 파일 수정 (겹치면 충돌이 나고, 충돌 해결은 모두의 시간을 먹는다)
- `main` 브랜치에 직접 푸시
- API 키를 코드에 붙여넣기 (키는 서버 환경변수로만, 통합 담당이 관리)

## 폴더 안내

```
api/       서버 함수 (Upstage 호출, 추천 계산) - 파트 A·D
public/    화면 파일 - 파트 B·C·D
scripts/   학식 자동 수집 - 파트 A
db/        DB 스키마 - 파트 A
docs/      API 계약, PR 가이드
```
